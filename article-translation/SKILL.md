# Перевод технической статьи (чат-версия конвейера)

Композитный скилл: стандарты перевода, политика терминов и словари в одном
процессе для работы в чате. Этот файл — маршрутизатор: жёсткий минимум и
порядок шагов здесь, детали — в `references/` (открывай по карте ниже, а не
все сразу).

**Границы применения.** Скилл — для перевода готовой статьи, присланной
пользователем. В чате нет изолированного трёхстадийного конвейера
(переводчик → редактор → проверяльщик) — его заменяют шаги 4–6 самопроверкой;
это честное ограничение, а не эквивалент полного конвейера.

## Жёсткий минимум (действует всегда, даже без открытия справочников)

1. **Блок-к-блоку.** Каждому блоку исходника (заголовок, абзац, список, код,
   таблица, цитата, картинка, сноска) соответствует блок перевода. Блоки не
   выкидываются, не сливаются и не добавляются; **лид** (текст между H1 и
   первым H2) — тоже блок, терять его нельзя. Единственное санкционированное
   добавление — сноска переводчика.
2. **Три исхода для термина:** распространённый — русское слово или
   устоявшийся англицизм инлайн; имя продукта / общеизвестная аббревиатура —
   чистый оригинал; узкоспециализированный — **сноска-определение** `[^N]` при
   первом упоминании с пометкой «— Прим. перев.» (нумерация продолжает
   авторские сноски). Пояснения в скобках и «обратный перевод» («композиции
   (compositions)») запрещены; скобки — только для расшифровки аббревиатур.
3. **Не переводить:** имена продуктов и компаний, код и идентификаторы,
   URL, названия статей в ссылках. Комментарии в коде — переводить.
4. **Смысл не редактируется.** Перевод не «улучшает» исходник: цифры, факты,
   утверждения и порядок аргументов — как у автора. Сомнение в термине —
   правило сомнения: не прошёл тест распространённости → сноска.
5. **Типографика русского текста:** кавычки «ёлочки», длинное тире, числа и
   даты по-русски. Пунктуация — по правилам русского языка, не исходника.
6. **Кальки чистятся системно**, а не по наитию: предскан по 9 категориям
   каталога (лексические, of-цепочки, вводные, there is/are, связки, пассив,
   модальность, местоимения-шум, цепочки существительных).

## Процесс

### Шаг 1. Вход

Получи статью (текст или URL — попроси вставить текст, если URL недоступен).
Определи домен (K8s / DevOps / сети / БД / ML-AI / безопасность / фронтенд /
мобильная разработка) и открой словарь домена из карты ниже; смежные домены —
два словаря. Зафиксируй исходник: автор, площадка, дата, URL.

### Шаг 2. Предскан терминов

Выпиши термины статьи, которых нет в словаре домена и Общей таблице. Для
каждого реши исход по тесту распространённости →
`references/translation-glossary.md`. Спорные — покажи пользователю списком с
предлагаемым исходом до перевода.

### Шаг 3. Перевод блок-к-блоку

Переводи литерально: тот же порядок блоков, картинки и сноски на местах,
формат заголовка и кавычек по стандартам →
`references/translation-standards.md`. Термины — по решениям шага 2 и словарю
домена. Длинная статья — переводи частями по разделам, не сокращая.

### Шаг 4. Чистка калек

Пройди перевод по 9 категориям каталога калек с протоколом правки →
`references/translation-calques.md`. Регистр: художественные украшения →
нейтральные, статусы систем — принятым в домене словом (stuck → «завис»).

### Шаг 5. Терминология и консистентность

Сверь термины со словарём домена и Общей таблицей →
`references/translation-glossary.md`: один термин — один перевод по всему
тексту; пояснение — на первом упоминании, не на втором. Склонение
англицизмов и падежи — по глоссарию.

### Шаг 6. Самопроверка

Блочный счётчик: посчитай блоки исходника и перевода по типам — числа обязаны
сойтись (авторские сноски 1:1; сноски переводчика — отдельной строкой). Лид на
месте. Чек-лист перед выдачей → `references/translation-standards.md`; защита
чисел и ссылок → `references/final-trust-editing.md`; пунктуация →
`references/technical-punctuation.md`.

### Шаг 7. Сдача

Отдай перевод + служебную сводку: атрибуция исходника (автор, площадка, дата,
URL), блочный счётчик, список сносок переводчика, спорные термины и их исходы.
Расхождение счётчика не прячь — покажи и объясни.

рантайму — в чате их заменяют шаги 4–6.
