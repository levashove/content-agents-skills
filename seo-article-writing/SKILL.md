# Написание SEO-статьи (чат-версия конвейера)

Композитный скилл: собирает редполитику SEO-домена и trust-оптимизацию в один
процесс для работы в чате. Этот файл — маршрутизатор: жёсткий минимум и порядок
шагов здесь, детали — в `references/` (открывай по карте ниже, а не все сразу).

**Границы применения.** Скилл — для написания статьи с нуля. Для аудита
cite-safety уже готового текста используй отдельный скилл
`seo-trust-optimization`. В чате нет изолированного фактчекера и машинных
детекторов репозитория — их заменяет самопроверка по шагам 5–7; это честное
ограничение, а не эквивалент полного конвейера.

## Жёсткий минимум (действует всегда, даже без открытия справочников)

1. **Цифры и факты — только с источником.** Полная атрибуция = первоисточник +
   год + URL (для мнений — ФИО/должность/компания). Запрещены «по данным
   экспертов», «исследования показывают» и цифры из агрегаторов без
   первоисточника. Не нашёл подтверждение — не пиши цифру или ставь маркер
   `[NEEDS_FACTCHECK: что проверить]`.
2. **Факты о продукте — только из фактуры пользователя.** Возможности, цены,
   лимиты, сертификации продукта не выдумываются и не «вспоминаются»: нет в
   предоставленных материалах — спроси или пометь `[NEEDS_FACTCHECK]`.
   Расхождение брифа с известными фактами — флаг, а не молчаливый выбор.
3. **Свежесть.** У быстро стареющих данных (объёмы рынка, доли, версии ПО,
   активность ransomware-групп) проверяй год; цифра старше 12 месяцев — только
   с явной пометкой года в тексте.
4. **Без keyword stuffing.** Плотность основного ключа 1–2%, читаемость важнее
   плотности. Ключ — в H1, первом абзаце, 1–2 H2 и заключении, не в каждом
   предложении.
5. **Extractable-принцип.** Первые 2–3 предложения каждой H2-секции —
   самодостаточные: их можно процитировать без остального текста (так статью
   цитируют AI-поисковики).
6. **Без ИИ-паттернов.** Запрещены hedge-зачины («В современном мире», «Важно
   понимать»), параллелизмы «не просто… а», пустые связки («Таким образом»),
   триады прилагательных ради ритма. Инфостиль: цифры и факты вместо общих слов.
7. **Один H1 в теле.** Варианты заголовков — только в обсуждении с
   пользователем, в текст идёт один выбранный.

## Процесс

### Шаг 1. Вход

Собери от пользователя: тему, основной ключ (если есть), аудиторию, цель,
фактуру продукта (если статья продуктовая). Чего-то нет — спроси одним
сообщением списком, не по одному. Без фактуры продукта продуктовую статью не
начинай.

### Шаг 2. Угол

Не проси угол у пользователя — синтезируй сам из темы и фактуры **3–5
вариантов угла** (тезис + обещание читателю в одно предложение) и дай выбрать.
«Обзор всего обо всём» — не угол. Детали и тезис-карта →
`references/editorial-angle.md`.

### Шаг 3. Структура

Предложи скелет: H1 (с ключом), H2 каждые 300–500 слов, списки/таблицы, FAQ
при информационном интенте. Требования к структуре, мета-данным (title до 60,
description 140–160) и LSI → `references/seo-baseline.md`.

### Шаг 4. Текст

Пиши по выбранному углу и утверждённой структуре. Тон эксперта →
`references/expert-writing.md`, пороги качества и стоп-слова →
`references/content-quality.md`, терминология продуктов →
`references/brand-terminology.md`.

### Шаг 5. Самопроверка источников

Пройди по каждой цифре и утверждению: атрибуция полная? первоисточник, не
агрегатор? год на месте? Общие требования →
`references/source-attribution.md` и `references/freshness-check.md`, иерархия
Tier → `references/source-standards.md`. Для темы ИБ дополнительно —
`references/cybersecurity-claims.md` (белый список Tier 1), для темы
LLM/агентов/AI-инфраструктуры — `references/ai-ecosystem-claims.md`
(версии и GA-статусы — только из release notes).

### Шаг 6. Чистка

Прогони текст по `references/ai-pattern-cleanup.md`: зачины, связки,
параллелизмы, утечки кухни (упоминания брифа/ТЗ/структуры в публикуемой
прозе). Числа, ссылки и атрибуции при чистке не трогаются — полный контракт
защиты → `references/final-trust-editing.md`.

### Шаг 7. Trust-блоки

Переработай ключевые секции в Extractable Trust Blocks по
`references/seo-trust-optimization.md`: самодостаточные цитируемые блоки,
cite-safety, FAQ под AI-выдачу. Специфика ранжирования Яндекса →
`references/yandex-specifics.md`, доступ AI-краулеров (если обсуждается
публикация на своём домене) → `references/ai-crawler-access.md`.

### Шаг 8. Сдача

Отдай текст + отдельным блоком: выбранный угол, meta title/description, список
источников, оставшиеся маркеры `[NEEDS_FACTCHECK]` (если есть) и что человеку
проверить руками. Не выдавай текст с маркерами за готовый к публикации.
