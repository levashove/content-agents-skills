# Редстандарты VK Tech

Это официальное руководство по стилистике и оформлению текстов VK Tech. **ОБЯЗАТЕЛЬНО** применяй эти правила при создании любого контента.

### Бренд-архитектура и нейминг (канон)

Три уровня продуктовой архитектуры VK Tech. Полная карта и фактура — «vk-tech» (внутренний материал системы, в эту выгрузку не входит).

1. **Направления** — Облачная платформа, Дата-сервисы, Сервисы продуктивности, Бизнес-приложения. Уровень для рассказа о портфеле в целом.
2. **Продукты и платформы** — основной уровень коммуникаций: английский, краткое имя, приставка **`VK`** (`VK Cloud`, `VK Data Platform`, `VK HR Tek`).
3. **Внутриплатформенные продукты** — **без приставки `VK`**, упоминаются **только в связке с платформой**.
   - Облако и Дата — по-английски: `Load Balancer`, `WAF`, `Cloud Servers`, `Trino`, `Kafka`, `Jatoba`.
   - Продуктивность и Бизнес-приложения — по-русски: `Почта`, `Диск`, `Документы`, `Календарь`, `КЭДО`, `Командировки`.

| Правильно | Неправильно |
| :---- | :---- |
| Календарь VK WorkSpace | VK Календарь, VK Calendar, «Календарь» в одиночку |
| Почта VK WorkSpace | VK Почта, VK Mail |
| Load Balancer в VK Cloud | VK Load Balancer |

**Визуальное:** один логотип — VK Tech. Названия продуктов — текстом; иконки продуктов только как навигационные элементы, не как отдельные лого.

## Названия мессенджеров и соцсетей

### Основные правила

- **ВКонтакте, Одноклассники** — кириллицей
- Если оригинальное название дано латиницей, сохраняем латиницу: **Twitter, Telegram, YouTube, Clubhouse, TikTok, Viber, WhatsApp**
- Исключения — **Инстаграм\*** и **Фейсбук\*** (по юридическим причинам)

### Личные аккаунты

Если речь идёт о персональном аккаунте или блоге: «в моём фейсбуке\*», «в его инстаграме\*», «в твиттере компании».

### Составные слова

- телеграм-канал (строчными, как ютуб-канал; с прописной — только в начале предложения)
- ютуб-канал
- инстаграм-аккаунт\*
- твиттер-профиль
- фейсбук-аккаунт\*
- дискорд-сервер
- Тиктоки, тиктокер, тикток-блогер

### SMM

**Эсэмэмщик** (SMM-специалист). Не «сммщик», не «SMM-щик».

### Stories и Reels

- **Stories** — название раздела
- **сторис, инстаграм-сторис\*** — единичные публикации
- **Reels** — название сервиса
- **рилс, рилсы** — ролики

## Сложные слова

### Несогласованные определения (латиницей)

Если определение состоит из одного слова, ставим дефис: *Spark-приложение, Docker-контейнер, JSON-файл*.

### Английские слова

Пишем с **прописной (большой)** буквы всегда (название вакансии, должности, технологии), кроме случаев, когда принято написание со строчной: *application-aware*.

Исключение: традиционные феминитивы — «учительница», «официантка», «предпринимательница».

## Типографика

### Общее оформление документа

> **Только для документов Google Docs / DOCX** (брифы, согласования,
> материалы «на вычитку»). К Markdown/CMS-каналам (блог, Хабр, Дзен, VC.ru,
> Telegram) шрифты/кегль/«Стили» неприменимы — там действует типографика
> ниже (числа, кавычки, тире) и правила канала.

- Шрифт — **Arial**
- Кегль — **11-й**
- Интервалы между абзацами — через кнопку «Интервал» → «Добавить интервал после абзаца»
- Межстрочный интервал — **1,15**
- Заголовки оформляем через «Стили»

### Числа

| Правило | Пример |
| :---- | :---- |
| Интервалы — среднее тире | 400–500 |
| Пробелы между разрядами (от 4 знаков) | 4 000, 40 000, 4 000 000 |
| Десятичные дроби — через запятую | 1,5 (не 1.5) |
| Миллиарды, миллионы | млрд, млн (без точки) |
| Тысячи | тыс. (с точкой) |
| Без сокращения, когда нет чисел | «потратил много миллионов» |
| Перед процентом — без пробела | 50% |
| Числа до 10 в косвенных падежах — словом | «до четырёх дней» |
| Числа до 10 в ряду других — цифрой | «11 предприятий, 8 офисов и 2 500 сотрудников» |
| Порядковые числа — с наращением | 2-е место, второе место |
| «Стопроцентный» — без наращения | 100% хлопок |
| Приблизительные числа — словами | «около двухсот подписчиков» |
| Исключение | 24/7 |

### Списки

#### Нумерованные списки

Используем, когда важна последовательность. **Цифры с точками**, пункты с большой буквы, заканчиваем точкой.

```
Как разморозить курицу:
1. Достаньте курицу из морозилки.
2. Положите в микроволновку.
3. Включите режим разморозки.
```

#### Маркированные списки

Используем, когда последовательность не важна. Маркеры — точки.

**Если пункт НЕ целое предложение:** с маленькой буквы, заканчиваем точкой с запятой, в конце последнего — точка. Исключение: в текстах VK WorkSpace «;» не используется — пункты-не-предложения без конечного знака, последний с точкой (оверлей «vk-workspace-style» (внутренний материал системы, в эту выгрузку не входит), он строже общего).

```
Чтоб украсить ёлку, нужны:
- дождик и мишура;
- безопасные гирлянды;
- ёлочные игрушки.
```

**Если пункт — целое предложение:** с большой буквы, заканчиваем точкой.

### Ссылки

- Ставим **на логическое словосочетание** с предлогом, не на одно слово
- Если ссылка в конце предложения — **точку не включаем**
- Кавычки в ссылку **не входят**
- **Не ставим** ссылку на слова «здесь», «тут»

| Неправильно | Правильно |
| :---- | :---- |
| отмечают в [HeadHunter] | [отмечают в HeadHunter] |

### Эмодзи

- **Отбиваем пробелом** от соседних слов
- Не выделяем курсивом
- После эмодзи в конце предложения **точку не ставим**
- В вопросительном/восклицательном — знак перед эмодзи: «Вы слышали об этом?» 😲

### Тире и дефис

| Тип | Использование | Символ |
| :---- | :---- | :---- |
| Длинное тире | Знак препинания | — |
| Среднее тире | Между числами | – |
| Дефис | Внутри слов | - |

#### Правило минимизации длинного тире

Длинное тире — частая болезнь AI-текстов: модели ставят его через предложение, и текст выглядит монотонно. **Сокращай количество длинных тире до минимума.**

**Где длинное тире обязательно:**
- Между подлежащим и сказуемым, выраженными существительными: «VK Cloud — облачная платформа»
- В значении «это», «это есть»: «Kubernetes — стандарт оркестрации контейнеров»
- Перед обобщающим словом после перечисления: «Серверы, хранилища, сети — всё это доступно в облаке»

**Лимит и замены — единый источник, здесь не дублируем:** числовой лимит длинных тире на 1000 знаков — в `references/brand-terminology.md`; таблица функциональных замен (вводная → запятые/скобки, пояснение → двоеточие, присоединение → точка, противопоставление → запятая с союзом) — в «technical-punctuation» (внутренний материал системы, в эту выгрузку не входит).

### Подписи к картинкам

- Выравнивание по левому краю
- Точку в конце **не ставим**
- Выделяем **курсивом**
- Кегль **10-й**

### Кавычки

#### В кавычках

- Названия из трёх слов, из двух — по ситуации
- Однословные названия, если непонятно значение: «Аммоний» (предприятие и вещество)
- Названия на русском с родовым словом: предприятие «Завод»

#### Без кавычек

- Аббревиатуры
- Названия на русском без родовых слов (кроме случая выше)
- Названия на английском: VK Tech

#### Нейминг VK + NAME

Вторая часть с большой буквы, склоняется: «в VK Работе», «на VK Объявлениях». По умолчанию без кавычек.

#### Типы кавычек

- Обычно — **ёлочки** («»)
- Внутри закавыченного текста — **лапки** („")

### Прехедеры

Без точек.

### Валюта

- В прозе предпочтительно **слова**: «рублей»
- Знак (₽, $) — в коротких форматах (рассылки, уведомления, СМС), таблицах и графике
- Порядок: число + знак через **неразрывный пробел**: «207 ₽», «1000 $»
- **Нельзя**: «₽ в месяц», «₽/мес.» — единицу времени писать словом: «207 ₽ в месяц»

### Объёмы данных

| Единица | Сокращение |
| :---- | :---- |
| Гигабайты | ГБ |
| Гигабиты | Гбит |
| Терабайты | ТБ |
| Мегабайты | МБ |

Байт — прописная **Б** (ГБ, ТБ, МБ); бит — строчная (Гбит, Мбит). Это снимает путаницу байт/бит.

### Математические знаки

- Минус: **−** (не среднее тире)
- Деление: **÷** или **:** (не слеш)
- Умножение: **×** (не буква х)
- Вокруг знаков — **пробелы**
