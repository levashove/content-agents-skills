# Правила написания с экспертной специализацией

Общие правила для всех экспертных специализаций SEO-модуля. Загружаются автоматически.

## Роль эксперта в генерации

Эксперт определяет:
- **Терминологию** — какие термины использовать, какие запрещены
- **Глубину** — насколько технически детальный текст
- **Фокус** — на какие аспекты темы делать акцент
- **Аудиторию** — кто читает (developers, architects, CTO, бизнес)

Эксперт НЕ определяет:
- Стиль (стиль — из `content-quality`)
- SEO-параметры (из `seo-baseline`)
- Типографику (из `brand-terminology`)

## Принципы экспертного контента

1. **Конкретные цифры** — производительность, метрики, SLA, а не «высокая надёжность»
2. **Практические примеры** — реальные сценарии, а не теория
3. **Сравнения** — с альтернативами, но без маркетинговых преувеличений
4. **Российский контекст** — сертификации, импортозамещение, локальные кейсы
5. **Актуальность** — версии продуктов, даты релизов проверяй через WebSearch

## Работа с датированными данными

Экспертные скиллы содержат справочные данные (кейсы, сертификаты, статистику). Эти данные:
- **Используй как отправную точку** — для понимания контекста и терминологии
- **Проверяй актуальность** через WebSearch — особенно даты, версии, цифры рынка
- **Не копируй слепо** — ресёрчер должен найти свежие данные

## Чего избегать

- Общие фразы без конкретики («инновационное решение»)
- Устаревшие факты без проверки
- Преувеличение возможностей продукта
- Игнорирование ограничений и trade-offs
- Прямая реклама вместо экспертного анализа

## Разметка рисков: [NEEDS_FACTCHECK]

Writer не обязан знать правильный ответ — но обязан помечать места, где может ошибиться. Это экономит ресурс фактчекера: он начинает с явных меток, а не вычитывает весь текст на предмет галлюцинаций.

### Когда ставить метку

Ставь `[NEEDS_FACTCHECK: что проверить]` на любом из следующих:

- Число, которого нет дословно в конспекте ресёрчера
- Цитата, дата, должность, место работы человека
- Версия ПО, релиз, EOL-дата
- Регуляторный документ (закон, приказ, ФЗ, ПП)
- Кейс компании с конкретными метриками (X% экономии, Y часов downtime)
- GitHub stars, MAU/DAU, доля рынка, темпы роста
- Прогнозы и «через N лет» утверждения

### Формат метки

```
В 2025 году рынок ИИ в РФ достиг [NEEDS_FACTCHECK: 168 млрд ₽ — TAdviser, дата] 168 млрд ₽.
Helm используют [NEEDS_FACTCHECK: 75% — CNCF Annual Survey, год] 75% организаций.
```

Метка инлайн, рядом с фактом. Фактчекер либо подтверждает (удаляет метку), либо корректирует и указывает в отчёте.

### Запрет писать без метки

Если в конспекте ресёрчера нет прямой цитаты числа — его **нельзя** вставлять без `[NEEDS_FACTCHECK]`. Даже если «кажется правильным».

## SEO-ключи в экспертных скиллах

Каждый скилл содержит списки primary/secondary/long_tail keywords. Это:
- **Отправная точка** для автоподбора ключей на шаге 4 wizard'а
- **Не жёсткое ограничение** — дополняй через WebSearch-анализ конкурентов
- **Основа для LSI** — генерируй LSI-ключи на базе этих списков
