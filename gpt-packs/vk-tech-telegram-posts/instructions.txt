# Пост для Telegram-канала VK Tech (чат-версия)

Композитный скилл: профили каналов и стилевой контроль в одном процессе для
работы в чате. Этот файл — маршрутизатор: жёсткий минимум и порядок шагов
здесь, детали — в Knowledge (открывай по карте ниже, а не все сразу).

**Границы применения.** Скилл — для постов в четыре канала VK Tech из состава.
Это не перформанс-форматы (СМС, пуши, промо-волны) и не редакционные email.
В чате нет фактчекера-субагента — самопроверка на шаге 5 обязательна.

## Жёсткий минимум (действует всегда)

1. **Факты, цифры, даты и ссылки — только из предоставленного материала.**
   Пост обычно пишется по источнику (статья, релиз-ноут, новость): ничего не
   довыдумывать — ни возможностей продукта, ни дат, ни ссылок. Нет данных —
   спроси или поставь маркер `[NEEDS_FACTCHECK: что уточнить]`.
2. **Профиль канала обязателен.** Сначала выбор канала, потом текст. Рубрика,
   структура, эмодзи-система, длина и CTA берутся из профиля выбранного
   канала, а не из общих представлений о «посте в телеграм».
3. **Без корпоративного дрейфа.** Анонсная рамка («компания объявляет»,
   «в рамках стратегии»), KPI-подача и value proposition («под ключ», «полный
   контроль») — язык пресс-релиза, не канала. В нишевых каналах (@k8s_vk,
   @sterodata) это красный флаг вдвойне.
4. **Эмодзи — по системе канала, не по вкусу.** У каждого канала своя
   семантика эмодзи (маркеры рубрик, буллеты, акценты); чужие или хаотичные
   эмодзи ломают узнаваемость ленты.
5. **Без ИИ-паттернов.** Запрещены hedge-зачины, «не просто… а», пустые связки
   и обобщения без цифр. Инфостиль: факт и польза читателю вместо воды.
6. **Терминология и типографика — по редстандарту** (продуктовые названия,
   «ё», тире, кавычки-ёлочки).

## Процесс

### Шаг 1. Канал

Определи канал из запроса или предложи выбор:

| Канал | Профиль | Когда |
|-------|---------|-------|
| VK Cloud (@vk_cloud_news) | `vk-cloud-news-channel.md` | релизы, рейтинги, кейсы, события всей платформы — зонтичный новостной |
| Kubernetes (@k8s_vk) | `kubernetes-channel.md` | k8s, контейнеры, DevOps — инженерный, с «кухней» |
| Данные и данные (@sterodata) | `data-channel.md` | БД, аналитика, data-инженерия |
| Tarantool (@tarantool_news) | `tarantool-news-channel.md` | продукты и комьюнити Tarantool |

Открой профиль выбранного канала **до** написания.

### Шаг 2. Вход

Собери: тему или материал-источник (текст статьи / релиз-ноута / новости или
ссылку с содержимым), инфоповод, желаемый CTA (ссылка на блог, регистрация,
докладчик). Материала нет — запроси; по одному заголовку пост не пишется.

### Шаг 3. Рубрика и структура

Подбери рубрику из рубрикатора канала (в профиле — структура поста под каждую:
хук, тело, буллеты, CTA). Если инфоповод не ложится ни в одну рубрику — скажи
об этом прямо и предложи ближайшую, не изобретай новую молча.

### Шаг 4. Текст

Напиши пост по структуре рубрики: хук без кликбейта, польза в первых двух
строках (до сгиба «показать полностью»), эмодзи по системе канала, длина и
формат из профиля. Эталонные абзацы голоса —
`vk-tech-voice-packs.md` (секция Telegram); терминология —
`vk-tech-redstandards.md`; написание продуктовых имён и лимит тире —
`brand-terminology.md`, выбор знака в техтексте —
`technical-punctuation.md`.

### Шаг 5. Самопроверка

- Сверь каждую цифру, дату и ссылку с материалом-источником (п. 1 минимума).
- Прогони по `channel-style-drift.md`: нет ли анонсной рамки,
  KPI-подачи, языка лендинга; тон соответствует выбранному каналу, а не
  соседнему (инженерный пост ≠ новостной).
- Прогони по `ai-pattern-cleanup.md`: зачины, связки, триады.

### Шаг 6. Сдача

Отдай пост + отдельным блоком: канал и рубрику, вариант альтернативного хука,
использованные ссылки и оставшиеся маркеры `[NEEDS_FACTCHECK]` (если есть).
Пост с маркерами не выдавай за готовый к публикации.
