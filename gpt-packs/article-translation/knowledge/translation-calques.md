# Кальки: каталог и протокол правки редактора

Часть стандартов перевода (этап 2, редактор; вынесено из `translation-standards.md`
для читаемости). Каталог калок с английского/немецкого по 9 категориям и протокол
правки. Общий процесс и принципы редактора — в `translation-standards.md`.

## Кальки с английского/немецкого: каталог и протокол правки (этап редактора)

Кальки — главная типовая проблема перевода (выявлено в анализе отчётов 2026-Q2). Переводчик работает дословно (это правильно — этап 1), а редактор обязан системно вычистить кальки на этапе 2. Это **не «по желанию»**, а основная задача редактора наравне с разбиением длинных предложений.

### Протокол правки калек (для редактора)

1. **Pre-scan.** Перед началом правки прогони текст через 9 категорий ниже и составь короткий список калек, которые будешь править (с цитатой «было/стало»). Если калек > 30 — это сигнал, что переводчик дал слишком сырой текст; пометь это в отчёте.
2. **Правка по категориям**, в порядке приоритета: лексические → конструкции `of` → вводные обороты → there is / there are → союзы и связки → пассив → модальность → местоимения → цепочки существительных.
3. **Не уходи в авторский пересказ** — режь только то, что калькировано. Если предложение читается естественно — не трогай.
4. **Документируй в отчёте** — счётчик правок по каждой категории (см. формат отчёта в `translation-editor.md`).

### Каталог калек по категориям

#### 1. Лексические (one-to-one замены)

| Калька | Замена | Источник в оригинале |
|--------|--------|----------------------|
| является / является чем-то | опустить связку, конкретный глагол | is / are |
| позволяет / даёт возможность | конкретный глагол: ускоряет, упрощает, открывает | allows / enables |
| осуществляет / производит | делает / выполняет / проводит | performs / carries out |
| представляет собой | опустить или «это» | represents / is |
| обладает (свойствами) | имеет / есть | possesses / has |
| данный / данное / данная | опустить (это калька артикля the) | the (как указательный) |
| такой / такая / такое | конкретное существительное или опустить | such |
| определённый | конкретный или опустить | certain |
| следующий (в значении «вот этот») | без «следующий», просто двоеточие | the following |
| вышеупомянутый / вышеописанный | «выше мы …» / опустить | aforementioned / above-mentioned |

Пример:
```
✗ «Данное решение является инновационным продуктом, которое позволяет
  осуществлять обработку данных».
✓ «Это решение обрабатывает данные».
```

#### 2. Конструкции с «of» / родительными цепочками

Английский легко строит цепочки `the X of the Y of the Z` — в русском такое читается тяжело.

| Калька | Замена |
|--------|--------|
| «реализация механизма аутентификации пользователей системы» (4-членная) | разбить на 2 предложения или поменять порядок: «реализуем аутентификацию — механизм работает так:» |
| «использование возможности конфигурации параметров» | «настраиваем параметры» |
| цепочка из 3+ родительных подряд | разбить или переформулировать с глаголом |

Правило: **не более 2 уровней родительного падежа подряд** в одной фразе.

#### 3. Вводные обороты (избыточные, выкидываются целиком)

| Калька | Замена |
|--------|--------|
| Следует отметить, что… | опустить |
| Важно отметить, что… | опустить или «важно: …» |
| Стоит сказать, что… | опустить |
| Необходимо понимать, что… | опустить вводную, оставить суть |
| Как было упомянуто выше… | «выше мы …» или ссылка на раздел |
| Как мы видим, … | опустить |
| По сути … / По существу … | опустить |
| Фактически … | опустить (это лжевводное, см. russian-grammar) |
| В сущности … | опустить |
| Можно сказать, что … | опустить |
| В то же время следует учитывать, что … | «но» / «однако» |
| Не секрет, что … | опустить |
| Очевидно, что … | опустить (если не противопоставление) |

Пример:
```
✗ «Следует отметить, что важно понимать, что данный подход
   позволяет улучшить производительность».
✓ «Этот подход ускоряет систему».
```

#### 4. There is / There are («есть» в начале предложения)

| Калька | Замена |
|--------|--------|
| Существует / есть несколько причин этого | «у этого несколько причин» / «это объясняется тем-то» |
| Имеется возможность сделать X | «можно сделать X» |
| Есть необходимость в X | «нужен X» / «понадобится X» |
| Существует проблема X | назвать проблему напрямую: «проблема: X» |

#### 5. Союзы и связки (избыточные обороты)

| Калька | Замена |
|--------|--------|
| Для того, чтобы | «чтобы» |
| Несмотря на тот факт, что | «хотя» |
| В связи с тем, что | «потому что» / «из-за» |
| По причине того, что | «потому что» |
| В соответствии с | «по» / опустить |
| По отношению к | «к» / «для» / опустить |
| Что касается … , то | вынести подлежащее в начало без «что касается» |
| В дополнение к этому | «кроме того» / «ещё» |
| Также, как и | «как и» / «и» |
| Таким образом | «значит» / «итог» / опустить |
| В результате | «в итоге» / опустить |
| Помимо всего прочего | «ещё» / опустить |

#### 6. Пассив → актив (где уместно)

| Калька | Замена |
|--------|--------|
| «Данные обрабатываются системой» | «Система обрабатывает данные» |
| «Файл был создан пользователем» | «Пользователь создал файл» |
| «Это будет рассмотрено далее» | «Рассмотрим это далее» / «Дальше — про это» |
| «Может быть использовано для» | «Используют для» / «Применяют для» |
| «Должно быть отмечено, что» | опустить (см. категорию 3) |

Пассив сохраняется, если он смысловой (научный стиль, неизвестный субъект): «Файл был создан 1 января» — нормально.

#### 7. Модальность (избыточная)

| Калька | Замена |
|--------|--------|
| Вы должны / Вам следует (в каждой инструкции) | прямой императив: «настройте», «откройте», «запустите» |
| Мы рассмотрим / Мы обсудим (в начале каждого абзаца) | «рассмотрим», «дальше — про это», убрать «мы» |
| Давайте рассмотрим | «рассмотрим» |
| Можно было бы сказать, что | опустить |
| В случае, если | «если» |

#### 8. Местоимения и указатели (вычистить шум)

| Калька | Замена |
|--------|--------|
| «Это / эти / то / те …» в каждом предложении | конкретное существительное или опустить (если ссылка на предыдущее очевидна) |
| «Тот же самый X» | «тот же X» / опустить «самый» |
| «Один из X» (если контекст и так ясен) | без «один из» |
| «Каждый из них», «все из них» | «каждый» / «все» (без «из них») |

#### 9. Длинные цепочки существительных и составные определения

Английский легко строит `user authentication mechanism implementation`. В русском это превращается в нечитаемый набор родительных падежей. Правка: вынести часть в глагол, разбить на два предложения, использовать дефис.

| Калька | Замена |
|--------|--------|
| «реализация механизма аутентификации пользователей» | «реализуем аутентификацию пользователей; механизм работает так:» |
| «процесс обработки запросов пользователей системы» | «как система обрабатывает запросы» |
| длинное составное прилагательное-калька | разбить на «X для Y» или использовать дефис |

### Регистр: художественная лексика и слова-статусы (вне 9 категорий)

Не калька, но родственный дефект перевода (разбор редакторских комментариев 2026-07-22):

- **Художественные наречия и метафоры-украшения** в техтексте — «незаметно стал», «тихо завоевал», «подарок операторам». Стилистический маркер оригинала (quietly, silently), не несущий смысла, при редактуре опускается или заменяется нейтральным: «стал стандартом де-факто», «удобен операторам». Если метафора авторская и смысловая — сохраняется (контракт литерализма это не отменяет: снимаем только украшение, не мысль).
- **Слова-статусы систем** — глаголы состояния (stuck, hang, freeze, fail) переводятся принятым в домене словом, не первым словарным значением: notebook is stuck → «notebook завис» (не «застрял»), the job hangs → «задача повисла». Сомнение — сверься с разделом глаголов доменного словаря или пометь `🔁 EDITOR_RESHAPE`.

### Чек-лист редактора: что проверить после правки

- [ ] В тексте нет «является», «позволяет», «осуществляет», «представляет собой», «данный» (как калька артикля)
- [ ] Нет «следует отметить», «важно отметить», «стоит сказать», «как мы видим», «по сути», «фактически» в качестве вводных
- [ ] Нет «существует» / «имеется» в начале предложения как калька «there is»
- [ ] Нет «для того, чтобы», «в связи с тем, что», «несмотря на тот факт, что»
- [ ] Не более 2 родительных падежей подряд в одной фразе
- [ ] Пассив сохранён только там, где он смысловой
- [ ] Нет «давайте рассмотрим» / «мы рассмотрим» в начале каждого блока
- [ ] Местоимения «это / эти / тот» — на месте, а не как калька артикля

Если хотя бы один пункт не выполнен — пройди по тексту ещё раз.
