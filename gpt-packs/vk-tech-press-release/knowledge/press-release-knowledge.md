## ФИРМЕННАЯ СТРУКТУРА VK TECH 2026

Выведена из реальных релизов на vk.tech (апрель–май 2026). Это основной образец — не путать со старым VK Cloud-форматом.

```
Дата
↓
Заголовок (H1)
↓
Лид — КУРСИВОМ, 1-2 предложения, с ГЛАВНОЙ ЦИФРОЙ   [ОБЯЗАТ.]
↓
Тело — 2-4 абзаца сплошняком, механика решения      [ОБЯЗАТ.]
↓
Цитата спикера — ОТКРЫВАЕТСЯ РЫНОЧНОЙ ЦИФРОЙ          [ОБЯЗАТ. для большинства]
↓
[Вторая цитата — партнёр / эксперт]                  [опц.]
↓
Заключительный факт-абзац — сертификаты/условия/сроки [ОБЯЗАТ.]
↓
[Бойлерплейт]                                        [опц.]
↓
[«Почитать по теме» — 3 связанные публикации]         [опц.]
```

### Три жёстких фирменных требования

1. **Лид всегда курсивом** (`*текст*`), сразу после заголовка, 1-2 предложения, с главной цифрой релиза.
2. **Цитата спикера открывается сильнейшим релевантным тезисом** — стратегической рамкой, значимостью новости или релевантной цифрой, затем проблема → решение → ценность. Рыночная цифра аналитика — один из вариантов, не обязательный (`press-release-standards.md` §2). Исключение по форме — релиз-мероприятие (цитаты организаторов).
3. **Заключительный факт-абзац** — отдельный финальный абзац: сертификации (ФСТЭК, 21 приказ, Tier 4, реестр ПО), условия участия, сроки, доступность.

### Фирменная концовка цитаты

После рыночной цифры и описания барьера цитата часто закрывается формулой **«мы закрыли / обнуляем этот барьер»**: *«…компании опасаются делегировать агенту реальные действия. Мы закрыли этот барьер изолированным контуром…»*. Это узнаваемый приём VK Tech — используй.

### Версии под разные СМИ

Частая практика: один релиз в двух версиях — **general** (деловые СМИ, меньше терминов) и **tech** (IT-СМИ, стек и стандарты: OWASP, УЗ-1…УЗ-4, архитектура). Если пользователь упоминает разные площадки — предложи сделать обе версии.

## ЗАГОЛОВКИ

### Формула

```
[Компания] + [глагол действия] + [продукт/суть] + [для кого / зачем]
```

| Контекст | Глаголы | Пример |
|----------|---------|--------|
| Новый продукт | запустил, представил, выпустил | VK Tech запустил сервис … VK AI Researcher |
| Обновление | обновил, расширил, улучшил | VK Tech расширил возможности размещения облачных ресурсов |
| Партнёрство | и [партнёр] будут / договорились | VK Tech и «ЦТиП» будут развивать ИТ-инфраструктуру для промышленности |
| Линейка | представил линейку, вывел на рынок | VK Tech вывел на рынок продукты информационной безопасности |
| Мероприятие | описательный | Хакатон … прошёл одновременно в 10 вузах |

До 100 символов, без превосходных степеней и восклицательных знаков. В режиме «варианты» автор даёт 3-4 заголовка с разным фокусом (продукт / ценность / цифра / партнёр).

## ЛИД-АБЗАЦ (курсивом, с цифрой)

### Структура

Лид — **всегда курсивом**, 1-2 предложения, с **главной цифрой** релиза. Три рабочих типа:

- **Короткий** (1 предложение — суть + ценность): *«Решение помогает компаниям работать с агентами в защищённом контуре.»*
- **Развёрнутый** (2 предложения — что + главная цифра + детали): *«VK Tech поддержит средний и малый бизнес … на общую сумму 500 млн рублей. Все участники получат грант 150 000 бонусов …»*
- **От контекста**: *«В дата-центре VK появилась четвёртая зона доступности — аттестованная площадка VK Cloud …»*

Главная цифра в лиде — это «крючок»: сумма, доля, количество, срок, порядковый номер («четвёртая зона»), кратность роста.

## ОПИСАНИЕ ПРОДУКТА (тело)

После лида — 1-3 абзаца сплошным текстом: что делает → как работает (термины расшифровать) → для кого / сценарии → варианты поставки. Для нескольких продуктов — по абзацу на каждый.

Расшифровывай термины для журналистов: `SDN — программно-определяемая сеть`, `CI/CD — процесс автоматической сборки и доставки кода`, `S3 — объектное хранилище для файлов любого типа`.

## ЦИТАТЫ СПИКЕРОВ

### Формула (открывается сильнейшим релевантным тезисом)

```yaml
1_strongest_thesis: "Сильнейший релевантный тезис: стратегическая рамка / значимость новости / релевантная цифра (рыночная цифра аналитика — вариант, НЕ обязанность — standards §2)"

3_solution:         "Как продукт снимает барьер"
```

## ЗАКЛЮЧИТЕЛЬНЫЙ ФАКТ-АБЗАЦ

Фирменный финал. Отдельный абзац после цитаты с конкретикой:

- **Сертификации:** «аттестован по 21 приказу ФСТЭК России и соответствует уровню Tier 4», «включён в реестр российского ПО», «соответствует требованиям 152-ФЗ по УЗ-1».
- **Условия и сроки:** «Заявки принимаются до 31 июля 2026 года на сайте VK Cloud», «доступен новым пользователям VK Cloud».
- **Доступность поставки:** «доступен в публичном облаке VK Cloud, а также в формате Private Cloud».

## «ПОЧИТАТЬ ПО ТЕМЕ» (опц.)

3 связанные публикации в конце релиза. Формат каждой карточки:

```
{дата} • {Заголовок связанной публикации} • Читать статью
```

Использовать, если есть релевантные релизы. По умолчанию — опционально (в мультивыборе блоков).

## РАСШИФРОВКА ТЕРМИНОВ ДЛЯ ЖУРНАЛИСТОВ

### Облако и инфраструктура

| Термин | Расшифровка |
|--------|-------------|
| IaaS | аренда вычислительных ресурсов — серверов, хранилищ и сетей |
| PaaS | готовая платформа для разработки с настроенным окружением |
| SaaS | готовое ПО, доступное через интернет по подписке |
| SDN | программно-определяемая сеть — управление трафиком на уровне ПО |
| Public / Private / Hybrid Cloud | публичное / частное / гибридное облако |
| On-Premise | установка ПО на собственных серверах клиента |
| Зона доступности | изолированная площадка дата-центра с собственным питанием и сетью |
| Tier 4 | максимальный уровень надёжности дата-центра по классификации Uptime |

### Данные, ИИ и базы

| Термин | Расшифровка |
|--------|-------------|
